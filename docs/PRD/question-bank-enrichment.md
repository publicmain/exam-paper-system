## 1. 现状（实测数据，2026-06-02 生产库）

全库 **1500 题**，但按"能不能自由印进可售卷"分，真实可用量极小：

| sourceType | 数量 | 能否可售出卷 |
|---|---|---|
| `past_paper_reference` | **1110 (74%)** | ❌ 版权真题，**只存元数据、无原文/多无答案**，`restricted_internal` 默认排除 |
| `ai_generated` | 352 (23%) | ⚠️ 取决于是否已审（见下）|
| `original_school` | **48 (3%)** | ✅ 校本原创，版权干净、可售 |

`ai_generated` 里关键的一笔：**`ai_quick_paper` = 291 道**，是"一键 AI 出卷"的副产物，**从未逐题审核 → 默认排除**出出卷池。这是本 PRD 最重要的反面教材：**没有质量门的 AI 量产，只会制造 291 道没人敢用的废题。**

**可售/可出卷（approved + active + 非 ai_quick）实测**：英语 363、雅思 483、物理 28、计科 33、9709 数学 22(含新作 10)、4024 数学 11、4MA1 数学 0。

**结论**：英语/雅思已厚；理科与数学薄；**自有版权的核心资产只有 48 道**。"1500 题"是虚数。

## 2. 目标与约束

- **目标**：极大扩充**可售**（版权干净 + 已审 + 可抽题）的题库，支撑多学科、多 component 的出卷，服务 SaaS。
- **约束**：
  1. **零-API（现状）**：运行时不调 LLM；量产 AI 需付费（= overhaul PRD 的 Phase 3 触发点）。
  2. **版权**：不得用真题原文；可售池只收**原创**或**授权可再分发**内容。
  3. **质量**：每道题须正确 + 过结构校验 + 过审计，否则不入可售池。
- **非目标**：把 1110 道版权真题"洗白"成可售内容（做不到，也不该做）。

---

## 3. 核心判断：瓶颈是"质量门"，不是"生成"

报告（Vibe→可售）的灵魂在此：**扩库的瓶颈从来不是能不能生成，而是生成完谁来把关。** 291 道废题就是证据。

> 所以无论用哪条生成源（AI / 老师 / 导入），都先穿过**同一道质量门**才进可售池。**门是第一性的，源是可插拔的。**

---

## 4. 三条生成源（诚实权衡）

| 源 | 能否上"极大"量 | 版权 | 质量风险 | 成本 | 角色 |
|---|---|---|---|---|---|
| **AI 量产** | ✅ 唯一能上量 | 干净(原创) | 高(必须靠门拦) | 需付费运行时 AI | **主力**（Phase 3 后）|
| **老师贡献** | 🟡 随客户复利 | 干净(校方拥有) | 中(同行评审) | 低 | SaaS 护城河 |
| **手工原创(我)** | ❌ 单点瓶颈 | 干净 | 低(逐题核验) | 低(我的 chat 时间) | 打样/补高价值缺口 |
| **授权采购** | ✅ 瞬间上量 | ⚠️ 许可受限 | 低(专业) | 高+许可谈判 | 可选加速 |

---

## 5. 推荐架构：内容质量门（无论用哪条源）

```
源(AI量产 / 老师写 / 人工导入)
  → 自动 critic（evaluator-optimizer：题面/答案/难度/题型自检）
  → groundedness / 答案正确性校验
  → 结构校验器（已有 paper-structure-validator）
  → 去重（同题/近似题）
  → 人工复核队列（teacher/admin approve）
  → 入可售库（approved_internal + 出处/版权标签）
```
这正是报告 Template C(critic) + evaluator-optimizer + groundedness gate 落到**内容生产**上。出处/版权标签全程携带，决定一道题能否进可售池。

## 7. 分阶段路线图

### Phase A ｜ 建门 + 盘活存量（**纯零-API、立即可做、零新增成本**）
- 建/补齐"内容质量门"流水线（§5）+ 人工复核工作流（产品已有 ReviewService/Review 页，验证并补齐）。
- **用现有 291 道 `ai_quick_paper` 当第一批输入**穿过这道门：合格的复核提升入可售库，不合格的淘汰。→ **不花一分 API 钱就能深挖存量、同时把门验真。**
- 我继续手工补高价值缺口（如各 component 起步批），走同一道门。

### Phase B ｜ AI 量产（触发条件：付费运行时 AI / overhaul Phase 3）
- 打开 AI 生成，**全部穿过 Phase A 的门**才入库；监控单位合格题成本。

### Phase C ｜ 老师贡献 + 多校共享（SaaS）
- 老师写题→同门审核入库；跨校共享池随客户增长复利，形成护城河。

---

## 8. 度量

- 可售题量 / 每学科·component（目标：每 component 足以出 N 份不重样卷）。
- 门的吞吐 + **人工复核拦截率**（catch rate）+ 误批率。
- 单位"合格入库题"成本（含 API + 人工）。
- 出卷"凑满目标分"成功率（直接受益指标）。

---

## 9. 风险与护栏

- **垃圾入库**（291 废题重演）→ 门优先、源穿门。
- **正确性**（错答案进可售卷）→ critic + 人工 gate + 抽检。
- **版权**（真题/授权内容泄入可售池）→ 出处标签 + 可售池只收原创/可再分发。
- **零-API 被动破**（量产必须付费）→ Phase B 触发条件量化，先把 Phase A 的零成本价值吃满。
